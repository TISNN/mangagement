# 导师管理中心（Mentor Management Center）设计文档

## 1. 模块定位与目标

- **所属导航**：`Starfleet Academy / Mentor Council`
- **角色覆盖**：导师、顾问、运营负责人、管理员。
- **核心目标**：
  - 建立导师全生命周期管理体系：档案、能力标签、排班、任务、绩效。
  - 支撑跨项目的导师调度，并与学生服务、任务看板、服务进度联动。
  - 提供能力评估、匹配推荐与AI辅助，提升导师资源利用率与协同效率。

## 2. 顶层信息架构

```
导师管理中心
├─ 导师总览（Mentor Overview）
├─ 导师名册（Mentor Ledger）
├─ 能力矩阵（Skill Matrix）
├─ 排班与可用性（Availability Planner）
├─ 任务与绩效（Mission & Performance）
├─ 导师详情（Mentor Profile）
└─ 数据洞察（Mentor Insights）
```

### 2.1 导师总览（Mentor Overview）
- 指标卡片：在册导师、活跃导师、平均利用率、即将过载导师、满意度。
- 过滤器：角色（文书/材料/顾问/质检/面试官）、语言、科目、区域。
- 人力负载雷达：按项目阶段（材料/文书/网申/面试）展示导师负载。

### 2.2 导师名册（Mentor Ledger）
- 列表卡片视图，展示导师头像、角色、擅长领域、负责学生数、活跃状态。
- 快速操作：指派任务/学生、发起沟通、查看排班、发送提醒。
- 支持收藏、标签筛选、自定义排序。

### 2.3 能力矩阵（Skill Matrix）
- 维度：服务类型（文书/材料/网申/语言等） × 评分（经验、评分、满意度、响应速度）。
- 扩展维度：语言能力、科目、国家申请经验、时间可用段。
- 支持AI推荐：根据学生需求自动推荐导师组合。

### 2.4 排班与可用性（Availability Planner）
- 周/月视图：展示导师的可用时间、已排任务/会议。
- 支持拖拽调整排班、批量导入（如Excel/Google Calendar）或API同步。
- 可设置“请假”“黑名单时间”“提醒周期”。

### 2.5 任务与绩效（Mission & Performance）
- 任务视图：来自项目任务控制台的导师任务列表，支持状态筛选、批量操作。
- 绩效指标：任务完成率、平均响应时长、学生反馈得分、退回率、AI建议采纳率。
- 可生成导师绩效报告，与薪酬/激励体系对接。

### 2.6 导师详情（Mentor Profile）
- Tab 结构：`概览 | 能力标签 | 任务 | 排班 | 历史服务 | 反馈 & 满意度 | 绩效报告 | 档案`
- 概览：基础信息、角色、当前项目、活跃状态、AI标签。
- 能力标签：专业方向、擅长语言、服务类型评级、可接项目类型。
- 任务：当前任务列表、历史任务、待完成项、相关学生。
- 排班：日历形式展示下周安排，可直接修改。
- 历史服务：关联学生/项目、服务评分、成果案例。
- 反馈 & 满意度：来自学生/顾问的评价，AI总结趋势。
- 绩效报告：按月/季度展示指标、亮点、改进建议。
- 档案：合同、证书、培训记录、合规信息。

### 2.7 数据洞察（Mentor Insights）
- KPI：导师利用率、任务完成率、学生满意度、风险提示。
- AI预测：识别潜在过载导师、缺少导师的服务类型、培训需求。
- 报告导出：支持PDF/Excel，供管理层决策。

## 3. 核心功能详解

### 3.1 导师档案与标签
- 标签体系：角色、语言、科目、服务方向、国家经验、时区、评级。
- 标签管理：可配置标签分类与等级，支持AI自动打标签（基于历史服务数据）。
- 档案完整度评分，提醒缺失信息（资格证书、合同、培训等）。

### 3.2 匹配与指派
- 基于学生需求（语言、专业、服务类型、时间段）生成导师推荐列表。
- 点选推荐后自动创建任务/连线学生，与项目任务控制台联动。
- 提供“组合推荐”：例如文书主导师 + 质检导师 + 面试导师。

### 3.3 排班与提醒
- 导师可设置可用时间，系统自动计算可调度时段。
- 当导师可用时段不足时发出预警，建议新增导师或调整任务。
- 支持短信/邮件/IM提醒即将开始的任务/会议。

### 3.4 绩效跟踪
- 指标包括：任务数量、平均响应时间、学生满意度、文书退回率、材料通过率。
- 支持角色对比（同类型导师之间的分析）。
- 链接到培训建议：若某项指标低，推荐相关课程或指导。

## 4. 数据模型（简化）

```
mentors
├─ mentor_profile (基础信息、能力标签、资质)
├─ mentor_skills (服务类型、等级、评分、语言)
├─ mentor_availability (时间段、可用状态)
├─ mentor_assignments (任务、学生、项目)
├─ mentor_feedback (学生/顾问反馈)
├─ mentor_performance (指标历史)
└─ mentor_documents (合同、证书、档案)
```

关键字段说明：
- `mentors`: id, name, primary_role, status(在岗/请假/停用), timezone, contact_info。
- `mentor_skills`: skill_type, level(1-5), experience_years, preferred_projects。
- `mentor_availability`: day_of_week, start_time, end_time, availability_status。
- `mentor_assignments`: assignment_type(任务/学生), linked_id, status, deadline, workload。
- `mentor_performance`: metric_name, metric_value, period, trend。

## 5. API 参考（示例）

- `GET /mentors`: 支持筛选角色、语言、技能、可用性。
- `GET /mentors/:id`: 返回导师详情（含档案、标签、任务）。
- `POST /mentors`: 新增导师（连同档案、标签）。
- `PATCH /mentors/:id`: 更新档案、状态、标签。
- `GET /mentors/:id/availability`: 获取排班信息。
- `POST /mentors/:id/availability`: 更新可用时间。
- `GET /mentors/:id/performance`: 获取绩效指标。
- `GET /mentors/recommendations`: 根据学生需求输出推荐导师。
- `GET /mentors/analytics`: 输出统计数据、负载分析。

## 7. 交互与体验

- 名册卡片支持拖拽至“收藏”或“重点关注”区。
- 排班日历提供拖拽调整、双击新建可用时段、冲突提示。
- 任务列表支持按导师/项目分组，批量指派或转移。
- 导师详情的AI区域提供洞察与建议，可一键转换为任务或提醒。
- 支持暗色模式和响应式布局，移动端保留核心功能。

## 8. 自动化与提醒规则

| 触发条件                                      | 动作                                                         |
|-----------------------------------------------|--------------------------------------------------------------|
| 导师即将超负荷（>80%）                         | 发出预警，建议重新分配任务或新增导师                         |
| 导师长时间未登录或未响应任务                   | 提醒导师，并通知顾问或运营                                   |
| 绩效指标连续下降                               | 推荐培训课程或安排1:1 复盘会议                               |
| 学生反馈评分 < 3                               | 自动通知顾问，与导师一同制定改进方案                         |
| 新增项目需匹配导师                             | AI 推荐合适导师组合，提供多方案选择                         |

## 9. KPI 与评估

- 导师利用率 ≥ 75%，过载导师比例下降。
- 平均响应时长 < 12h，任务准时率提升。
- 学生反馈满意度逐季提高。
- 导师培训完成率、能力提升情况可量化。
- AI 推荐使用率与准确率逐步提升。

## 10. 迭代路线

1. **MVP**：导师名册 + 能力标签 + 基础排班 + 任务列表。
2. **能力矩阵与AI推荐**：上线热力图与推荐算法。
3. **绩效与档案**：增加绩效报告、档案管理、合同提醒。
4. **自动化与提醒**：接入任务、反馈、排班的自动提醒规则。
5. **生态对接**：与外部日历、IM工具、HR系统对接，实现自动同步。
