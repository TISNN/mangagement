# 项目市场（Project Marketplace）设计文档

> **一句话定义**：项目市场是一个让机构与导师发布优质留学项目、学生顾问快速筛选并签约的共享生态，所有上架内容都经过审核，确保真实可靠。

## 1. 背景与目标
- 现有平台聚焦服务执行（学生管理、导师管理、服务进度），缺少项目供给侧的“集市”入口，导致顾问难以洞察机构/导师的特色项目。
- 构建项目市场能将“服务供给”标准化，帮运营团队沉淀可反复销售的产品包，支撑多国、多专业、多价位的组合。
- 目标：为机构和导师提供安全、易用的上架通道；为顾问和学生提供可比较、可成交的项目列表；为运营提供审核与数据监控工具。

## 2. 角色画像与价值主张
| 角色 | 主要目标 | 在项目市场中的关键动作 | 核心价值 |
|------|----------|------------------------|----------|
| 机构管理员 | 扩大项目曝光，获取学生报名 | 提交项目、维护档案、回应审核意见 | 获得精准客户线索，缩短签约周期 |
| 导师/顾问 | 展示个人精品服务或小班产品 | 申请上架、更新项目动态、维护口碑 | 拓展收入渠道，累积教学信用 |
| 内部审核员 | 把控内容质量与合规性 | 审核项目、打回复审、下架违规内容 | 保证市场可信度，降低纠纷 |
| 平台顾问 | 为学生匹配最合适项目 | 搜索筛选、收藏、发起咨询或合同 | 提高转化效率，提供专业建议 |
| 学生/家长 | 快速了解可选项目 | 浏览、对比、申请顾问回访 | 省时省心找到适配项目 |

## 3. 核心功能模块
1. **项目上架工作台**（机构/导师视角）  
   - 模板引导式表单：基础信息、服务亮点、价格套餐、办理周期、成功案例、FAQ。  
   - 资质上传：营业执照/导师认证、合同模板、退费规则。  
   - 草稿保存、提交审核、查看审核状态与意见。
2. **审核后台**（内部视角）  
   - 待审核列表、项目详情比对、风险标签（价格异常、资质缺失、投诉记录）。  
   - 审核动作：通过/打回（带备注）/下架/标记整改。  
   - 审核记录留痕、自动提醒提交方。
3. **项目展示大厅**（顾问、学生视角）  
   - 卡片式项目列表，支持国家/阶段/服务类型/预算/语言等筛选。  
   - 项目详情页：服务流程、导师阵容、真实评价、可预约时间。  
   - 顾问快捷操作：收藏、生成推荐清单、发起签约流程。
4. **运营工具**  
   - 项目表现分析（浏览、收藏、咨询、签约、退费）。  
   - 风险监控面板：投诉率、审核通过率、违规类别分布。  
   - 内容管理：热点推荐位、专题策划、下架策略。

## 4. 用户流程（简化）
### 4.1 机构/导师上架
1. 登录后台 → 进入“项目市场 / 我的项目”。  
2. 选择“新建项目” → 根据步骤填写信息 → 上传资质。  
3. 提交审核后进入“待审核”状态，系统推送审核提醒。  
4. 审核通过：项目上线并可在大厅展示；审核打回：调整后再次提交。

### 4.2 审核员处理
1. 在“审核中心”查看待审核项目。  
2. 检查基础信息、价格、服务流程、资质。  
3. 审核结果可为：通过 / 打回修改 / 拒绝并列入黑名单建议。  
4. 审核结果同步给提交人，并记录到审计日志。

### 4.3 顾问与学生匹配
1. 顾问进入“项目大厅”，根据学生需求筛选项目。  
2. 查看详情 → 收藏或生成推荐清单 → 发送给学生。  
3. 顾问发起“咨询/签约”流程，自动关联 CRM 与服务交付模块。  
4. 学生确认后，生成正式服务合同，触发后续项目管理流程。

## 5. 信息架构（IA）
```
项目市场
├─ 我的项目（机构/导师）
│  ├ 草稿箱
│  ├ 待审核
│  ├ 已上线
│  └ 被下架
├─ 审核中心（内部）
│  ├ 待处理
│  ├ 已处理
│  └ 风险警报
├─ 项目大厅（顾问/学生）
│  ├ 推荐专题
│  ├ 分类筛选
│  └ 我的收藏
└─ 运营分析
   ├ 数据概览
   ├ 项目表现榜单
   └ 风险监控
```

## 6. 数据模型草案
```
project_market_item
  ├─ project_package (多个套餐)
  ├─ project_document (资质文件)
  ├─ project_audit_log
  ├─ project_tag_link
  └─ project_feedback (学生/顾问评价)
```
- `project_market_item`：id, owner_id, owner_type(机构/导师), title, country, education_level, service_category, price_range, duration_weeks, highlight_points(json), success_case(json), faq(json), status(draft/pending/online/offline), audit_score, created_at, updated_at。
- `project_package`：item_id, package_name, price, currency, deliverables(json), refund_policy, seat_limit, tutor_ratio。
- `project_document`：item_id, doc_type(license/certificate/contract), file_url, expire_at, verified_by。
- `project_audit_log`：item_id, auditor_id, action(submit/approve/reject/offline), reason, checklist_snapshot(json), created_at。
- `project_feedback`：item_id, from_user_id, rating, comment, case_reference_id, status(pending/published/blocked)。
- `project_tag_link`：item_id, tag_id (如“西班牙硕士”“德国双元制”“奖学金”）。

### 与现有系统的关键关联
- `owner_id` 对接 `organizations` / `mentor_profiles` 表，复用现有机构、导师档案。
- 审核通过后，可自动生成 `service_templates`（若已有表）供顾问一键创建学生服务。
- 签约后写入 `student_services`，保证交付模块与市场模块闭环。

## 7. 审核与风控机制
- **审核清单**：基础信息完整性、合法资质、价格与服务匹配度、过往投诉记录、退费规则、风险说明。
- **评分模型**：内容质量、资质可信度、成交表现、服务口碑；低分项目标记为“整改观察”。
- **风控策略**：  
  - 自动检测超低价/高价项目，触发人工复核。  
  - 非实名导师发布需先经过身份认证。  
  - 投诉率 >= 5% 或连续差评 >3 次，自动下线并通知运营。  
  - 审核日志与审批人责任人落地，支持追溯。

## 8. 关键页面交互要点
- **项目卡片**：显示国家、服务阶段、价格区间、评分、可预约时间。悬停出现“加入清单”“快速分享”按钮。
- **详情页**：采用分区布局——（1）基础信息；（2）服务流程时间轴；（3）导师/机构介绍；（4）过往成功案例；（5）套餐比较；（6）FAQ 与退费说明；（7）评价区。
- **审核界面**：分左右布局，左侧基础信息，右侧资质文件预览、审核清单、操作按钮，支持模板化意见。
- **上架表单**：拆分为多个步骤（基础信息 → 服务亮点 → 价格与套餐 → 资质与文件 → 确认），提供进度保存。

## 9. 指标体系（MVP 要追的 7 个数字）
1. 上架项目数（按国家/阶段拆分）。  
2. 审核通过率与平均审核时长。  
3. 顾问搜索使用率（含筛选项点击率）。  
4. 项目收藏/推荐清单转化率。  
5. 项目咨询转签约比例。  
6. 退费/投诉率。  
7. 单项目平均营收贡献。

## 10. 里程碑规划
| 阶段 | 时间 | 交付结果 | 备注 |
|------|------|----------|------|
| 0. 需求确认 | Week 0 | 设计文档、IA、数据模型 | 与运营/法务对齐审核标准 |
| 1. MVP | Week 1-4 | 上架表单、审核后台、项目大厅基础筛选 | 先支持机构管理员角色 |
| 2. 扩展版 | Week 5-8 | 套餐管理、顾问推荐清单、评价体系 | 接入导师个人项目 |
| 3. 优化 | Week 9-12 | 数据看板、A/B 推荐位、风控自动化 | 与 CRM 签约流程联动 |

## 11. 可能风险与应对
- **信息失真**：引入评分与评价机制，同时要求项目更新提醒。  
- **审核压力大**：设计分级审核（快速审核 + 复审），并提供批量操作。  
- **项目雷同**：提供模板化推荐字段，同时允许机构上传独特亮点；重复项目标记合并。  
- **法律合规**：与法务确认退款与合同模板，必要时增加免责声明。  
- **技术复杂度**：MVP 先使用现有 Supabase 表扩展；后续再拆分微服务。

## 12. 后续想法
- 结合 AI 推荐，根据学生画像自动推荐 TOP3 项目。  
- 引入项目对比功能，生成 PDF 推荐报告。  
- 与支付系统联动，实现在线订金/预付。  
- 项目市场与导师市场联动，支持捆绑套餐（项目 + 指定导师）。  
- 开放对外展示（企业官网市场页），作为品牌曝光渠道。
