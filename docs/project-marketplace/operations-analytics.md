# 运营工具与数据分析中心（Operations & Analytics Center）

> 目标：为运营、商务、风控团队提供一站式的项目市场数据监控、内容管理与策略调度能力，让项目生态稳定增长。

## 1. 模块定位
- **运营大屏**：实时掌握项目供给、曝光、转化情况。
- **内容控制枢纽**：配置推荐位、专题、标签，灵活调整市场氛围。
- **风控哨兵**：监控异常数据、投诉，触发自动下架或提醒。

## 2. 目标用户与需求
| 用户 | Core 需求 | 模块支持 |
|------|-----------|----------|
| 市场运营 | 提升项目曝光、策划活动 | 推荐位、专题管理、A/B 测试 |
| 商务经理 | 评估机构表现、制定合作策略 | 项目表现榜单、线索转化漏斗 |
| 风控团队 | 识别风险项目、处理投诉 | 风险监控面板、自动告警策略 |
| 内容编辑 | 维护标签、文案、专题页 | 资源库、内容版本管理 |

## 3. 功能结构
```
运营工具与数据分析中心
├─ 概览仪表盘
│  ├ 总项目数、上线率、待审核数
│  ├ 曝光 → 点击 → 咨询 → 签约 转化漏斗
│  ├ 投诉与退费速览
│  └ 今日提醒（即将下架、资质到期）
├─ 项目表现分析
│  ├ 按国家/阶段/机构的表现榜单
│  ├ 热门搜索词 & 筛选条件命中率
│  ├ 清单转化统计（顾问维度、学生维度）
│  └ 留存分析（项目复购/续签率）
├─ 推荐与专题管理
│  ├ 推荐位配置（Banner、卡片、排序）
│  ├ 专题策划（主题、项目集合、文案、有效期）
│  ├ 活动标签管理（限时折扣、奖学金等）
│  └ A/B 实验（多个版本曝光对比）
├─ 风险与合规监控
│  ├ 投诉处理流程（工单联动）
│  ├ 风险榜单（投诉率、退费率、低评分项目）
│  ├ 自动下线策略（阈值配置）
│  └ 审核工作量预测（根据趋势调整人力）
└─ 数据导出与 API
   ├ 周报/月报模板导出
   ├ 自定义维度数据导出
   └ 对接 BI 工具的 API（Looker Studio/Power BI）
```

## 4. 数据指标定义
- **曝光量（Impressions）**：项目卡片在大厅中被展示的次数。
- **点击量（Clicks）**：项目详情页被打开的次数。
- **咨询量（Leads）**：点击“预约顾问”或“咨询”按钮。
- **签约量（Deals）**：项目成功生成合同（签约状态）。
- **收藏量/清单加入量**：顾问或学生收藏/加入清单的次数。
- **投诉率**：投诉数量 ÷ 签约量。
- **退费率**：退费订单 ÷ 签约量。
- **审核通过率**：通过项目 ÷ 审核完成项目。
- **上线活跃率**：过去 30 天有曝光的项目占比。

## 5. 核心流程
### 5.1 推荐位配置
1. 运营进入推荐位管理 → 选择展示区域（首页 Banner / 主题卡等）。
2. 选择项目或专题 → 编辑文案、有效期、优先级。
3. 提交后实时生效，并在项目大厅前端展示。
4. 系统记录曝光、点击、转化数据，供后续分析。

### 5.2 风险处理
```
投诉/异常数据触发
   ↓
风控面板生成告警卡片
   ↓
运营查看详情（投诉内容、项目数据）
   ↓
选择动作：标记为观察 / 暂停推荐 / 强制下线
   ↓
系统同步到审核后台与项目提交者
```

### 5.3 数据报告生成
- 运营选择周期（周/月/自定义）与指标组合 → 系统生成报告。
- 支持导出为 PDF/Excel，也可直接发送到管理层邮箱。
- 报告模板包含：整体指标、国家榜单、热门项目、风险摘要、下周建议。

## 6. 监控与告警策略
- **实时告警**
  - 投诉率超过 5% → Slack/邮件通知运营与商务。
  - 单项目退费金额超过阈值 → 自动标记高风险并暂停推荐。
  - 审核待办 > 48 小时未处理 → 通知审核主管。
  - 资质文件即将过期（提前 30 天提醒）。
- **趋势告警**
  - 搜索零结果率连续三天 > 10% → 提醒补充相应项目。
  - 某国家项目数量环比下降 30% → 提醒商务关注供给。
  - 推荐位点击率持续低于平均值 20% → 建议 A/B 调整。

## 7. 数据可视化建议
- **漏斗图**：曝光→点击→咨询→签约。
- **热力图**：国家 vs 阶段的项目数量与转化率。
- **时间线趋势**：日/周曝光、咨询、投诉曲线。
- **矩阵图**：机构表现矩阵（客户满意度 vs 成交额）。
- **地图视图**：按国家展示项目数量与热度。

## 8. 权限与审核机制
- 推荐位配置需两人复核：运营编辑 → 主管确认。
- 风险下线操作需要记录执行人、原因、证据截图。
- 数据导出需权限控制，敏感字段（如合同金额）默认脱敏。
- 日志保留：所有配置变更需保留 1 年可追溯记录。

## 9. KPI 目标
1. 推荐位点击率 ≥ 8%，专题转化率逐步提高到 12%。
2. 投诉处理平均耗时 ≤ 24 小时。
3. 风险自动识别准确率 ≥ 80%（人工复核确认）。
4. 项目供给稳定度：热门国家项目数量波动 ≤ ±10%。
5. 运营配置错误率（被回滚的配置） ≤ 1%。

## 10. 迭代路线
| 阶段 | 功能 | 说明 |
|------|------|------|
| MVP | 指标仪表盘 + 推荐位管理 + 投诉监控 | 支撑日常运营决策 |
| V1.1 | A/B 测试 + 专题活动配置 | 支撑活动运营 |
| V1.2 | 风险告警自动化 + 审核量预测 | 提升风控效率 |
| V1.3 | BI API + 自定义报表引擎 | 支撑管理层与跨团队协作 |
| V1.4 | AI 策略建议（补充项目、调价建议） | 智能化运营助手 |

## 11. 简化说明（给初中生）
- 想象自己是商店经理，这个系统就是“后台大屏”，告诉你哪些商品卖得好、哪些有投诉。
- 如果某个商品问题多，你可以一键把它下架或者打标签提醒顾客。
- 同时，你还可以决定哪些商品放在最显眼的位置，吸引更多人来看看。
