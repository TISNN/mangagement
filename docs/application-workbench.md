# 文书与申请工作台（Application Workbench）产品设计

## 1. 模块定位与目标

- **角色覆盖**：顾问、文书老师、学生、质检人员、管理员。
- **核心诉求**：
  - 提供端到端的材料、文书、网申协同空间。
  - 降低手工记录成本，提升流程透明度与可控性。
  - 构建可复用的模板、最佳实践和质控标准。

## 2. 顶层信息架构

```
文书与申请工作台
├─ 任务总览（Task Dashboard）
├─ 材料清单（Material Checklist）
├─ 文书工作台（Document Workspace）
├─ 网申进度（Submission Tracker）
├─ 审核质检（Quality Control）
└─ 资源模板库（Resource Library）
```

### 2.1 任务总览（Task Dashboard）
- 看板 + 时间轴视图汇总各项目进度。
- 提供学生、顾问、状态、截止日期等维度筛选。
- 自动突出“今日到期”“逾期”“待学生响应”等卡片。

### 2.2 材料清单（Material Checklist）
- 默认表格视图，可切换卡片视图。
- 每行字段：项目、材料名称、阶段、状态、责任人、截止日期、最近更新、附件、操作。
- 支持批量指派、批量状态更新、批量催办、导出。
- 侧边抽屉展示材料详情（历史记录、附件、评论）。
- 模板套用：按国家/专业快速生成材料列表。

### 2.3 文书工作台（Document Workspace）
- 顶部标签切换文书类型（PS、Essay、CV 等）。
- 工具区：版本历史、差异比较、评论、AI 助手、模板引用、下载。
- 富文本编辑器支持 Markdown 切换、自动保存、手动里程碑保存。
- 协同评论：选中文本留言，支持 @ 成员并转换待办。
- AI 助手：润色、降重、语法检查、逻辑建议，输出可选择性应用。
- 模板/片段库：快速插入结构、段落，保存最佳实践。
- 流程按钮：提交审核、请求学生反馈、标记定稿。

### 2.4 网申进度（Submission Tracker）
- 学校/项目列表 + 阶段进度条（账号创建→表单填写→材料上传→缴费→提交→补件→录取结果）。
- 每阶段记录负责人、完成时间、备注，支持上传证据。
- 逾期自动高亮并触发提醒，支持导出 PDF 报告。

### 2.5 审核质检（Quality Control）
- 列表筛选：审核对象类型、状态、审核人、评分。
- 审核界面：左侧预览文书/材料，右侧评分表单（维度可配置，如语言、内容、结构、格式等）。
- 支持模板化质检标准，退回需填写原因并指派。
- 数据统计：平均审核耗时、退回率、常见问题标签。

### 2.6 资源模板库（Resource Library）
- 分类：文书案例、写作指南、表格模板、录取案例。
- 标签与全文搜索，支持预览与引用。
- 统计引用次数、支持权限控制（内部/对学生开放）。

## 3. 核心流程

1. **项目初始化**：顾问创建申请项目 → 选择学校/专业 → 套用材料与文书模板 → 指派工作人员 → 设置截止日期。
2. **文书协同**：文书老师撰写 → 版本管理 → 评论与学生反馈 → AI 辅助 → 提交质检 → 定稿归档。
3. **材料推进**：顾问/学生上传材料 → 状态同步 → 触发提醒 → 完成后自动流转到下一阶段。
4. **网申执行**：记录账号、表单进度、缴费与提交 → 生成证据 → 补件提醒 → 录取结果记录。
5. **质检闭环**：审核模板配置 → 审核评分 → 退回/通过 → 统计报表输出。

## 4. 角色与权限

| 角色       | 权限亮点                                                                 |
|------------|--------------------------------------------------------------------------|
| 顾问       | 创建/管理申请项目、指派任务、推进网申、查看全局状态                      |
| 文书老师   | 编辑文书、接收任务、与学生协作、提交定稿                                |
| 学生       | 查看状态、上传材料、对文书反馈                                           |
| 质检人员   | 查看待审任务、评分与意见、退回修改                                       |
| 管理员     | 维护模板、配置质检标准、查看全量数据、权限管理                           |

## 5. 数据模型概览（简化）

- `application_project`：学生、学校、专业、状态、关键日期。
- `application_material`：关联项目，记录材料类型、状态、负责人、附件。
- `application_document`：文书对象，包含类型、当前状态。
- `document_version`：文书版本内容、作者、状态、AI 评分。
- `submission_progress`：阶段名称、状态、时间、证据。
- `quality_check`：审核结果、评分维度、审核人、意见。
- `template_resource`：模板、案例、片段，附标签与权限。

## 6. API 示例（REST）

- `POST /application-projects`：创建项目。
- `GET /application-projects/:id/materials`：获取材料列表。
- `PATCH /application-materials/:id`：更新材料状态或责任人。
- `POST /application-documents/:project_id`：创建文书类型。
- `POST /document-versions/:document_id`：新增文书版本。
- `POST /document-versions/:id/submit-review`：提交质检。
- `GET /submission-progress/:project_id`：查询网申阶段。
- `POST /quality-checks`：记录审核结果。
- `GET /resources`：查询模板库。

## 7. 交互与体验细节

- **时间线视图**：按时间展示关键节点与状态。
- **通知机制**：即将到期/逾期自动提醒，站内信+邮件/微信可配置。
- **协同评论**：支持 `@` 指定人员、转换待办、操作留痕。
- **AI 能力**：提供可配置 prompt、记录使用效果、支持文书评分。
- **筛选与搜索**：多维组合筛选，高效定位项目与任务。
- **安全与审计**：所有版本、审核、提交操作写入审计日志。

## 9. KPI 指标建议

- 文书平均审核通过率提升（目标 +X%）。
- 因材料缺项导致的延误率下降（目标 -Y%）。
- 顾问人均节省行政整理时间（目标 Z 小时）。
- 学生端满意度评分提升。
- 模板库引用次数与复用率提高。

## 11. 预期成功标准

- 提交前缺项率 < 5%，材料逾期率同比下降。
- 文书平均审改轮次减少，平均审核时长缩短。
- 顾问满意度评分提升，学生投诉率下降。
- 模块上线后 3 个月内，>80% 申请项目使用该工作台全流程。
