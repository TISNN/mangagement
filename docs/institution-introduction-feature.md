# 机构介绍功能方案

## 1. 背景与目标
- 现在机构没有一个集中展示“我是谁、能做什么、做出过哪些成绩”的入口，学生和家长得靠顾问口头介绍，效率低。
- 机构介绍功能要做成一个随时可更新的展示页，让机构能够一句话说明价值、用流程解释服务方式、用案例证明实力。
- 目标：帮助机构快速建立信任，让学生和家长在第一次接触时就能理解机构的特色、服务边界与成功经验。

## 2. 谁会用到
- **机构负责人 / 运营老师**：配置内容、上传案例、维护联系方式。
- **顾问团队**：把介绍页面作为沟通材料，快速讲清机构能力。
- **学生与家长**：自助了解机构，决定要不要进一步沟通或预约咨询。

## 3. 使用流程（对外展示视角）
1. 学生或家长打开机构介绍页（来自官网、分享链接或后台二维码）。
2. 先看到一句话定位、核心亮点，再向下浏览服务项目、流程说明、常见问题。
3. 如想进一步了解，可查看精选案例、团队介绍、常见疑问。
4. 最后点击“预约咨询”或“加入兴趣清单”，系统记录线索并通知顾问。

## 4. 页面主要模块
| 模块 | 目的 | 主要内容 | 备注 |
| --- | --- | --- | --- |
| 头图与一句话定位 | 3 秒内讲清机构价值 | 品牌 Logo、一句话介绍、核心数据（累计服务人数、成功率等） | 支持视频或轮播图 |
| 核心亮点卡片 | 告诉用户为什么选我们 | 3-4 张亮点卡片（师资、服务体系、成果、保障措施） | 每张卡片带图标和一句解释 |
| 服务项目总览 | 展示能提供的服务 | 服务分类卡片 + 价格区间 + 适用人群 | 支持“展开详情”查看更完整说明 |
| 服务流程 | 降低不确定性 | 从咨询到签约、服务执行、结项的步骤图 | 每步附上时长/责任人/交付物 |
| 成功案例库 | 用事实建立信任 | 3-6 个精选案例，包含学生背景、目标、成果、顾问点评 | 支持按服务类型筛选 |
| 团队介绍 | 展示专业度 | 核心顾问 / 服务团队合影、资历、擅长领域 | 支持跳转到导师详情页 |
| 学员/家长评价 | 社会证明 | 真实反馈、评分、感谢信截屏 | 可接入现有评价系统 |
| 常见问题（FAQ） | 回答高频疑问 | 8-10 个问题，覆盖服务流程、费用、退款、数据安全 | 可按主题折叠 |
| 预约与联系入口 | 转化 | CTA 按钮、咨询表单、电话、微信二维码 | 表单提交自动写入 CRM |
| 机构资质与合规信息 | 提升可信度 | 营业执照扫描件、办学许可证、合作认证、获奖证书 | 支持放大查看与下载 |
| 实体地址与服务网络 | 方便线下来访 | 总部地址、分校分布、地图、交通指引 | 集成地图组件，支持一键唤起导航 |
| 官方收款与账号安全 | 明确支付方式 | 对公银行账户、收款二维码（微信 / 支付宝）、发票开具说明 | 默认遮挡部分账号，点击验证身份后查看 |

## 5. 内容制作模板（给运营使用）
### 5.1 一句话定位模板
> 我们帮助【目标人群】在【时间范围】内完成【关键成果】，靠【核心方法/优势】实现稳定结果。

### 5.2 服务项目说明模板
- 服务名称（如“美国本科全程申请”）
- 适用人群：如“高中生，GPA 3.0+”
- 服务期限：如“12 个月”
- 交付内容：列出 3-5 个关键交付物
- 价格区间：如“¥80,000 - ¥120,000”
- 成功案例：链接到案例模块

### 5.3 案例模板
| 字段 | 示例 |
| --- | --- |
| 学生标签 | “高二 · 国际课程 · TOEFL 95” |
| 目标 | “冲刺 TOP30 工程专业” |
| 服务策略 | “暑研 + 机器人竞赛 + 文书深挖科研亮点” |
| 结果 | “录取 USC · 提供 50% 奖学金” |
| 顾问寄语 | “学生最大亮点是动手能力，我们重点展示动手项目” |

### 5.4 机构资质展示模板
- 证件名称：如“营业执照”“民办学校办学许可证”
- 发证单位：如“北京市海淀区市场监督管理局”
- 发证日期：如“2024-03-15”
- 证件编号：支持局部打码展示
- 证件图片：上传高清扫描件（系统自动加水印）
- 附加说明：如“认证有效期至 2027-03-14”

### 5.5 联系与收款信息模板
- 总部地址：具体到楼层（如“北京市朝阳区望京 SOHO T2 1508”）
- 服务时间：如“工作日 10:00-19:00”
- 联系方式：电话、邮箱、客服微信二维码
- 对公账户：开户名称、开户行、账号（自动打码显示前后 4 位）
- 微信/支付宝收款：展示二维码 + 收款账号昵称，支持切换户名
- 发票须知：开票资料、开票流程、预计寄出时间

## 6. 数据与配置建议
- **数据结构**
  - `institution_profile`: `id`, `tagline`, `mission`, `highlightStats`, `contactInfo`
  - `institution_services`: `serviceId`, `title`, `description`, `priceRange`, `duration`, `targetAudience`, `deliverables`
  - `institution_cases`: `caseId`, `coverImage`, `studentProfile`, `goal`, `strategy`, `result`, `advisorNotes`
- `institution_team`: `memberId`, `name`, `role`, `bio`, `avatarUrl`, `expertiseTags`
- `institution_credentials`: `credentialId`, `type`, `title`, `issuedBy`, `issueDate`, `mediaUrl`, `description`
- `institution_locations`: `locationId`, `name`, `address`, `contactPhone`, `businessHours`, `mapLink`, `transportGuide`
- `institution_payment_channels`: `channelId`, `type`(`bank`/`wechat`/`alipay`), `displayName`, `accountMasked`, `qrCodeUrl`, `bankName`, `notes`, `lastVerifiedAt`
- **内容管理**
  - 支持后台富文本编辑与图片上传，保存至 Supabase 存储。
  - 提供预览模式，避免草稿直接对外展示。
  - 内容发布需要二次确认，记录发布人和时间，方便追溯。

## 7. 交互与体验细节
- 头图区域背景可自动根据上传图片提取主色，保持品牌一致。
- 案例卡片支持左右滑动（移动端）或分页（桌面端）。
- FAQ 支持关键字搜索，输入“价格”即可跳到相关问题。
- “预约咨询”表单提交后展示成功提示，并提供继续浏览或返回首页按钮。
- 若机构暂未填写某个模块，前端展示“敬请期待”提示，保证页面完整。
- 营业执照等证件支持水印遮罩与下载权限控制；付款二维码在大屏展示时自动淡化，避免被偷拍。
- 收款信息在首次查看时需要手机验证码校验，并记录查看日志，防止外部恶意调用。

## 8. 埋点与监控
- 页面访问：记录访问来源、停留时间、滚动深度。
- CTA 点击率：统计“预约咨询”“拨打电话”“添加微信”等按钮点击次数。
- 案例阅读：记录每个案例被打开的次数，帮助运营挑选热门内容。
- 表单提交：记录提交成功率、字段缺失情况，出现异常时发送告警。

## 9. 发布计划（建议 3 周）
1. **第 1 周**：确认内容结构与设计稿，完成数据结构定义。
2. **第 2 周**：前端页面开发 + 后端接口/存储联调，加入埋点。
3. **第 3 周**：内部填充真实内容，完成测试与优化，灰度上线 1-2 家机构。

## 10. 风险与应对
- **内容更新不及时**：设置月度提醒，后台显示“内容已 30 天未更新”提示。
- **图片版权**：后台提示需上传自有或正版照片，增加版权确认勾选框。
- **案例隐私**：默认隐藏学生姓名，可由运营选择是否显示缩写或别名。
- **表单数据丢失**：提交失败时自动重试，并允许用户重新发送。

## 11. 下一步可扩展
- 支持多语言版本（中文/英文），方便双语宣传。
- 接入动态推荐：根据访问者需求（本科/硕士/语言）自动排序服务。
- 增加“资料下载”模块，让家长获取详细服务手册或流程 PDF。
- 与 CRM 打通线索评分，判断哪些访问者最关注某类服务，方便后续跟进。
