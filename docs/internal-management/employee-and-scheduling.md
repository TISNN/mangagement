# 团队成员中心（工作量视图）

## 1. 模块定位

团队成员中心用于全面掌握团队成员的基本信息、工作量与职责分配，帮助运营、人力及教务团队快速识别高负载成员与重点任务，保障交付质量与成员健康度。

## 2. 主要使用人群

- **教务/行政人员**：统筹导师、顾问的任务与会议安排，及时缓解资源冲突。
- **导师与顾问**：查看个人重点任务与会议安排，明确当前关注事项。
- **人力资源**：监测工作负载、超负荷风险与出勤表现，为绩效与排班决策提供依据。

## 3. 核心功能

- **成员档案管理**：集中展示基本信息、岗位、技能标签、所在团队与联系方式。
- **工作量看板**：实时汇总任务数量、会议压力与工作量指数，自动识别高负载成员。
- **职责清单**：展示每位成员的重点任务/会议（实时读取 Supabase 任务与会议 API），包含状态、优先级与截止时间。
- **高负载提醒**：当成员工作量 ≥ 80% 时即时发出提醒，提示协调资源。
- **出勤趋势**：保留近三个月的出勤表现与加班提醒，辅助排班与绩效复盘。

## 4. 数据结构与接口

| 表/实体 | 关键字段 | 说明 |
|---------|----------|------|
| `staff_profiles` | `id`, `name`, `role`, `skills`, `status`, `hire_date` | 员工档案 |
| `tasks` | `id`, `title`, `status`, `priority`, `assigned_to`, `due_date` | 成员任务 |
| `meetings` | `id`, `title`, `meeting_type`, `start_time`, `participants` | 成员会议 |
| `attendance_logs` | `id`, `staff_id`, `clock_in`, `clock_out`, `source`, `remark` | 出勤打卡 |
| `shift_requests` | `id`, `staff_id`, `type`, `status`, `approver_id`, `reason` | 请假/调班申请（保留，后续与工作量联动） |

工作量看板依托任务与会议数据，后续可与教育服务模块、排课系统（`scheduling-classroom.md`）联动，自动补齐课程与教室信息。

## 5. 典型业务流程

1. 教务或负责人在工作量看板查看整体负载，定位高负载成员。
2. 即时提醒提示当某成员工作量达到 80% 阈值，便于第一时间协调资源。
3. 进入成员详情页确认当前重点任务、会议与可用时间，必要时协调支援。
4. 若成员工作量持续高于阈值，可在任务中心拆分或改派任务，并记录调整结果。
5. 出勤趋势用于校验任务完成情况与出勤表现，辅助绩效与排班决策。

## 6. 监控与预警

- 高负载监控：工作量指数 ≥ 80% 时自动标红提示，并建议协调支援。
- 任务堆积：48 小时内到期的任务 ≥ 2 个时触发提醒，提示拆分或改派。
- 状态矛盾：成员状态为“请假/培训”但仍存在任务或会议时提示修正。
- 即时提醒：在界面内弹出 Toast 通知，提示高负载成员与关注事项，减少漏看风险。
- 出勤提醒：迟到/缺勤即时通知，异常累计达到阈值自动触发 HR 跟进。

## 7. 迭代建议

- 与第三方任务管理工具或企业微信任务 API 对接，减少手工录入。
- 提供成员自助标记工作重心的入口，确保重点任务信息及时准确。
- 引入 AI 工作量平衡建议，根据负载、任务类型和满意度历史优化资源调度。
