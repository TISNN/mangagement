# 合规监督中心

## 1. 模块定位

合规监督中心聚焦合同、隐私、财务与教学流程的风险控制，通过标准化检查表、取证记录与告警机制，帮助机构满足政策监管与合作伙伴要求，降低运营风险。

## 2. 主要使用人群

- **合规专员 / 法务人员**：制定检查项、审核合同、处理风险事件。
- **业务负责人**：查看所属项目的合规状态与整改进度。
- **管理层**：关注合规数据报表、重大风险预警。

## 3. 核心功能

- **检查计划**：按业务线创建周期性检查任务（招生流程、合同模板、数据隐私等）。
- **风险清单**：记录发现的问题、影响等级、整改负责人与截止日期。
- **合同与文档审查**：上传合同或协议，使用模板对比、风险提示。
- **政策速递**：追踪教育、签证、数据合规相关政策变动，推送解读。
- **告警与汇报**：重大风险触发短信/邮件告警，自动生成月度/季度合规报告。

## 4. 数据结构与接口

| 表/实体 | 关键字段 | 说明 |
|---------|----------|------|
| `compliance_programs` | `id`, `name`, `frequency`, `owner`, `scope` | 合规检查计划 |
| `compliance_checklists` | `program_id`, `item`, `standard`, `risk_level` | 检查项明细 |
| `compliance_cases` | `id`, `program_id`, `risk_level`, `status`, `assignee_id`, `due_at` | 风险事件 |
| `contract_reviews` | `id`, `contract_name`, `counterparty`, `status`, `issues` | 合同审查记录 |
| `policy_updates` | `id`, `title`, `source`, `summary`, `impact_area` | 政策动态 |

对外提供导入/导出接口，便于批量创建检查项；同时需要审计日志记录每次修改。

## 5. 典型业务流程

1. 合规专员建立“招生合同月度体检”计划，选定检查项和执行人。
2. 系统在月初自动生成检查任务，提醒执行人提交现场材料。
3. 检查过程中若发现风险，创建整改工单，指定负责人和期限。
4. 整改完成后上传佐证，合规专员审核并关闭工单。
5. 系统汇总未完成项、逾期率，自动输出合规周报/季报。

## 6. 监控与告警

- **风险看板**：按业务线展示风险数量、等级、整改进度。
- **逾期提醒**：整改任务逾期 24 小时以上自动升级告警。
- **政策推送**：关键政策变动通过站内信 + 邮件同时触达相关负责人。

## 7. 迭代建议

- 与合同系统、CRM 服务中心联动，自动识别即将到期或缺少附件的合同。
- 引入评分模型，对风险事件进行量化，便于横向对比不同业务线。
- 支持外部审计账号访问，只读查看相关材料与日志。
