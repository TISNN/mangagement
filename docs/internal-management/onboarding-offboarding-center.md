# 入职与离职中心（Team Onboarding & Exit Hub）

## 1. 模块定位

入职与离职中心用于统一管理团队成员的入职、转正、岗位调整与离职全生命周期流程。提供标准化任务清单、资料收集、审批流与跨部门协同能力，确保新成员快速融入、离职交接顺畅，并为数据沉淀与风控提供依据。

## 2. 主要用户

- **HR / 人才运营**：发起入离职流程、跟踪完成度、出具报表。
- **直属主管 / Buddy**：完成岗位培训、工具授权、知识传承。
- **IT / 财务 / 行政**：处理账号开通、资产领用、薪酬结算、工牌物品管理。
- **新成员 / 离职成员**：上传资料、签署协议、完成自检清单。

## 3. 核心功能

1. **流程模板管理**
   - 支持按职位类别（顾问、导师、后台）、工作地点（总部/远程）维护标准入职/离职模板。
   - 模板包含任务列表、责任人、预计耗时、并行/串行逻辑、提醒配置。
   - 提供版本控制，变更模板需管理员审核。

2. **入职流程执行**
   - HR 录入新成员基本信息，选择适用模板并指定入职日期、汇报对象、Buddy。
   - 自动生成跨部门任务，如账号开通、设备准备、培训安排。
   - 新成员端显示入职日程、资料上传、制度阅读确认。
   - 支持自动提醒（邮件/飞书/站内）与逾期预警。

3. **试用期与转正管理**
   - 在入职流程中绑定试用期目标、阶段评估表、360 反馈。
   - 到期前提醒主管完成转正评估，自动生成转正审批工作流。

4. **离职流程执行**
   - 支持主动离职、被动离职、试用期不合格等场景。
   - 自动生成交接清单（项目、客户、文档、账号、物品）。
   - 跨部门任务（财务结算、IT 回收、行政退卡、法务协议）统一管理。
   - 雇主品牌维护：离职面谈记录、满意度调研、校友群邀请。

5. **资料与签署管理**
   - 支持上传劳动合同、保密协议、证件扫描件，关联成员档案。
   - 集成电子签名服务（计划阶段可先模拟录入签署状态）。
   - 支持导出入职材料一览、离职面谈纪要。

6. **统计与报表**
   - 入职完成率、平均准备周期、离职交接完成率、试用期转正率。
   - 队列视图按部门/岗位显示当前待入职、待离职成员。
   - 提供月/季度报表导出 Excel / PDF。

## 4. 数据结构建议

### 4.1 主表

| 表名 | 核心字段 | 说明 |
| --- | --- | --- |
| `hr_onboarding_cases` | `id`, `employee_id`, `status`, `template_id`, `start_date`, `deadline`, `buddy_id`, `owner_id` | 入职流程实例 |
| `hr_offboarding_cases` | `id`, `employee_id`, `status`, `reason`, `last_work_day`, `template_id`, `handover_owner_id` | 离职流程实例 |
| `hr_process_templates` | `id`, `type`（onboarding/offboarding）, `applicable_roles[]`, `steps` JSONB, `version`, `is_active` | 流程模板 |
| `hr_process_tasks` | `id`, `case_id`, `step_id`, `title`, `assignee_id`, `status`, `due_at`, `completed_at`, `comments` | 流程任务 |
| `hr_documents` | `id`, `case_id`, `document_type`, `file_url`, `signed`, `signed_at` | 资料与签署 |

### 4.2 关联表

- `employees`：成员信息。
- `departments`：部门维度。
- `assets`：IT/行政资产台账（待后续对接）。
- `hr_surveys`：满意度与离职面谈记录。

## 5. 典型流程

### 入职流程
1. HR 发起 → 选择模板 → 填写基础信息 → 指定 buddy & 主管。
2. 系统生成任务 → 各责任人接收提醒 → 完成后打钩、上传附件。
3. 新成员登录后完成资料填写、制度确认 → 系统记录进度。
4. 到入职日完成开卡、账号激活 → HR 调度未完成任务。
5. 试用期到期前系统提醒 → 主管填写评估 → 提交转正审批。

### 离职流程
1. HR 创建离职单 → 选择离职原因、最后工作日、交接负责人。
2. 自动划分任务（项目交接、资产归还、账号关闭、离职面谈）。
3. 交接负责人上传交付清单，系统校验与项目记录一致性。
4. 财务确认结算，IT 关闭账号 → HR 审核完成 → 更新员工状态。
5. 可选步骤：发送离职关怀邮件、邀请进入校友群。

## 6. 权限与角色

- **HR 管理员**：配置模板、查看所有流程、审批异常。
- **HR 成员**：创建流程、跟进任务、导出报表。
- **部门主管**：查看本部门成员流程、处理指派任务。
- **Buddy/交接人**：查看被指派任务、提交反馈。
- **新成员/离职成员**：查看自助任务、上传资料（仅限个人流程）。

## 7. 指标与监控

- 平均入职准备时间、逾期任务数。
- 试用期转正及时率、转正通过率。
- 离职交接完成率、资产回收及时率。
- 满意度评分（入职第 30 天、离职面谈）。
- 异常预警：关键资料未签署、IT 账号未关闭。

## 8. 集成与扩展

- **集成**：对接日历、通知（邮件/IM）、电子签约、资产管理、薪酬系统。
- **扩展能力**：
  - 多语言支持（境外团队）。
  - API 输出流程状态给数据仓库。
  - 与绩效系统联动（入职目标、离职总结）。

## 9. 迭代建议

1. **MVP**：模板 + 任务清单 + 手动提醒 + 入离职报表导出。
2. **增强**：自动提醒、电子签署集成、试用期评估、自助门户。
3. **成熟阶段**：流程智能推荐、风险预警、跨系统自动化（薪酬/资产）。
