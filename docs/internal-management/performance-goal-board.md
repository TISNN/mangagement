# 绩效与目标看板（Performance & OKR Board）

## 1. 模块定位

绩效与目标看板集中管理团队及个人的目标（OKR/KPI）、关键结果、评估周期和反馈记录，形成“目标设定 → 执行跟踪 → 复盘评估 → 激励发放”的闭环。模块强调数据驱动与透明协同，帮助管理者及时发现偏差、成员清楚自己要达成的成果。

## 2. 用户角色

- **团队负责人 / 部门主管**：制定目标、拆解指标、跟踪执行、进行绩效评估。
- **个人成员**：查看目标、更新进度、提交成果、接收反馈。
- **HRBP / 绩效专员**：配置考核模板、管理评估周期、导出绩效报表。
- **跨部门协同者**：了解共享目标的进度，提供支持与确认。

## 3. 功能架构

1. **目标树与对齐**
   - 支持组织 → 部门 → 小组 → 个人四级目标树。
   - 目标可关联上级目标，展示对齐关系与贡献度。
   - 支持 OKR（目标 + 关键结果）和 KPI（数值目标）两种模式。

2. **目标设定与审批**
   - 提供目标设定向导：描述、指标、基线/目标值、权重、周期、负责人、协同人。
   - 设定完成后，走自定义审批流（如直属主管 → 部门负责人 → HR）。
   - 支持模板：如“业务开发”“顾问服务”“导师授课”等常见场景。

3. **进度更新与执行跟踪**
   - 成员可以录入关键结果的进度（百分比/数值）、上传佐证材料。
   - 系统支持从外部数据源（任务完成、工单数量、排班时长）自动更新部分指标。
   - 提供趋势图、红黄绿状态、里程碑提醒。

4. **评估与反馈**
   - 到期前触发评估任务：负责人自评、同级/下属/上级评价、HR 审核。
   - 评估结论支持打分、文字反馈、改进建议。
   - 可配置评级规则、权重、奖惩项，输出绩效结果。

5. **激励与发放记录**
   - 对接奖金/股权激励，记录发放金额、发放时间、对应目标。
   - 支持导出发放名单给财务或股权管理系统。

6. **看板与报表**
   - 首页 KPI：目标完成率、延迟目标数、优秀案例、风险目标。
   - 提供按部门/岗位/业务线拆分的绩效分布图、历史趋势图。
   - 支持导出 OKR 摘要、绩效评估报告。

## 4. 数据结构建议

| 表名 | 核心字段 | 说明 |
| --- | --- | --- |
| `performance_cycles` | `id`, `name`, `start_at`, `end_at`, `status`, `scope`, `owner_id` | 绩效/OKR 周期 |
| `performance_goals` | `id`, `cycle_id`, `parent_goal_id`, `title`, `type`（okr/kpi）, `owner_id`, `department_id`, `description`, `weight`, `status` | 目标主表 |
| `performance_key_results` | `id`, `goal_id`, `metric_name`, `baseline`, `target`, `current`, `unit`, `progress`, `auto_sync_config`, `last_synced_at` | 关键结果 |
| `performance_updates` | `id`, `key_result_id`, `updated_by`, `progress`, `evidence_url`, `comment`, `updated_at` | 进度更新 |
| `performance_reviews` | `id`, `goal_id`, `cycle_id`, `reviewer_id`, `score`, `feedback`, `review_type`（self/peer/manager/hr）, `submitted_at` | 评估记录 |
| `performance_rewards` | `id`, `goal_id`, `reward_type`, `amount`, `issued_at`, `issued_by` | 激励发放记录 |
| `performance_notifications` | `id`, `target_id`, `message`, `trigger_event`, `read` | 提醒通知 |

## 5. 指标与提醒

| 场景 | 规则 | 提醒对象 |
| --- | --- | --- |
| 目标进度低于 50% 且周期过半 | 自动标记黄色预警 | 目标负责人、直属主管 |
| 关键结果 7 天未更新 | 发送提醒 | 所有协同人 |
| 评估任务逾期 | 每日提醒 + 上级通知 | 评估责任人、HR |
| 目标完成率 100% | 发送祝贺/案例收集 | 团队负责人、人力 |

## 6. 权限设计

- **系统管理员**：配置周期、模板、审批流、评分规则。
- **部门负责人**：查看本部门目标、审批、给出反馈。
- **目标负责人**：创建/编辑自己目标、更新进度、发起协同。
- **协同人**：查看目标、更新自己负责的关键结果。
- **只读访问者**：跨部门查看特定目标（需要授权）。

## 7. 数据来源与集成

- 与任务管理（Task Management）联动：同步完成任务数量、响应时效。
- 与排班/出勤模块联动：获取实际授课时长、服务交付数据。
- 与 CRM 或项目系统联动：同步签约金额、客户满意度等业务指标。
- 与薪酬系统（未来）联动：将绩效结果影响奖金发放。

## 8. 迭代建议

1. **基础版**：手动录入 OKR/KPI、进度更新、简单评估流程、CSV 导出。
2. **增强版**：自动采集数据、可视化趋势、预警规则、激励记录。
3. **智能版**：目标建议库、自动权重分配、AI 生成评语、案例沉淀。

## 9. 页面交互思路

- 左侧目标列表 + 右侧详情面板，支持快速切换视角（个人/团队/组织）。
- 提供目标评分仪表、进度条、关键结果图表、评估时间轴。
- 成员端提供“我的目标”“待办更新”“收到的反馈”清单。
- 支持在团队成员详情页嵌入目标卡片，形成个人洞察视图。
