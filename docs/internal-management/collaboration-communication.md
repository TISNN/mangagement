# 协作与沟通记录（Collaboration & Communication Hub）

## 1. 模块定位

协作与沟通记录模块聚合团队内部公告、会议纪要、跨部门协作事项、任务跟进与评论动态，统一管理信息流与协作历史，提供提醒与归档能力，避免沟通内容分散在多个渠道导致遗漏。

## 2. 用户角色

- **团队负责人 / 项目经理**：发布公告、分配协作事项、跟踪推进状态。
- **团队成员**：接收通知、回复评论、记录会议纪要、提交成果。
- **跨部门协同人**：查看与自己相关的事项、同步信息、反馈进展。
- **HR / 行政 / 合规**：发布制度通知、收集反馈、留存记录。

## 3. 功能模块

1. **公告中心**
   - 支持发布全员或分组公告，数据结构包含标题、正文、附件、起止时间。
   - 提供已读/确认按钮，可查看阅读情况和提醒未读人员。
   - 模板支持：制度更新、节假日安排、战略重点等。

2. **会议纪要库**
   - 与会议模块联动，自动拉取会议基础信息。
   - 提供纪要模板（议题、决策、待办事项、责任人、截止日期）。
   - 支持附件上传、评论讨论、关键决策标记。
   - 可导出 PDF，支持权限控制（例如仅参会人员可见）。

3. **跨部门协作板（协同事项）**
   - 类似轻量级项目看板，记录协作事项、负责人、协作部门、状态、优先级。
   - 支持评论、附件、关联任务/目标。
   - 提供待办列表、流程式视图（待认领 → 进行中 → 待反馈 → 已完成）。

4. **沟通时间轴**
   - 汇总公告、会议、协作事项、任务评论、赞同/点赞等互动行为。
   - 成员可按对象（个人/部门/项目）查看，支持关键词搜索。
   - 提供过滤器：仅看 @ 我、仅看审批提醒、仅看未读。

5. **提醒与订阅**
   - 用户可订阅感兴趣的主题（如“导师团队”“文书项目”）。
   - 通过邮件/IM/站内信发送提醒，支持自定义提醒频率（即时/日结/周报）。
   - 提供钉板（Pin）功能，固定重要通知于顶部。

6. **知识沉淀与搜索**
   - 所有公告、纪要、协作事项可按标签、关键词、发布人搜索。
   - 支持智能推荐相关资料（后期与知识库集成）。

## 4. 数据结构建议

| 表名 | 核心字段 | 说明 |
| --- | --- | --- |
| `collab_announcements` | `id`, `title`, `content`, `audience_scope`, `tags[]`, `attachments[]`, `start_at`, `end_at`, `created_by`, `status` | 公告主表 |
| `collab_announcement_reads` | `id`, `announcement_id`, `user_id`, `read_at`, `confirmed` | 阅读/确认记录 |
| `collab_minutes` | `id`, `meeting_id`, `title`, `content`, `decisions[]`, `action_items[]`, `attachments[]`, `visibility_scope`, `created_by`, `created_at` | 会议纪要 |
| `collab_minutes_comments` | `id`, `minute_id`, `author_id`, `content`, `created_at`, `attachments[]` | 笔记 & 评论 |
| `collab_tasks` | `id`, `title`, `description`, `owner_id`, `department_id`, `status`, `priority`, `due_at`, `related_project`, `tags[]`, `watchers[]` | 协作事项 |

| `collab_subscriptions` | `id`, `user_id`, `topic`, `channel`（email/im/in-app), `frequency`, `last_sent_at` | 订阅配置 |

## 5. 指标与报表

- 情报覆盖率：公告阅读率、确认率。
- 协作效率：协作事项平均完成时长、逾期数量。
- 沟通活跃度：评论数、@使用次数、附件上传量。
- 会议成果：纪要提交率、行动事项完成率。
- 合规追踪：制度公告确认率、敏感公告访问审计。

可提供周/月报表导出，向管理层展示团队沟通运行情况。

## 6. 权限与可见性

- **公告权限**：管理员/HR/负责人可发布，设置可见范围。
- **纪要权限**：默认参会人员可见，可添加额外观众或限制访问。
- **协作事项**：创建者、负责人可编辑；关注者可评论；跨部门协作需考虑只读/编辑权限。
- **时间轴**：个人默认只看自己有权访问的内容。

## 7. 集成点

- 与会议管理（Meetings）同步会议基本信息、参会者列表。
- 与团队成员中心共享成员/部门数据，实现提醒推送、权限校验。
- 与知识库/文件中心关联，纪要与文档可互相跳转。
- 与 IM（飞书/企业微信）整合，实现消息推送与快捷回复。

## 8. 迭代路线

1. **MVP**：公告发布+阅读确认、会议纪要记录、协作事项看板、基础评论与提醒。
2. **进阶**：时间轴汇总、订阅与自定义提醒、模板化纪要、附件管理。
3. **高级**：智能推荐、自动摘要（AI）、跨系统消息汇聚（邮件转发、IM Webhook）。

## 9. 页面设计要点

- 主视图分为三个 Tab：公告、纪要、协作事项，右侧提供时间轴快捷视图。
- 公告列表支持置顶、标签过滤、阅读率显示；详情页突出确认按钮与附件。
- 纪要支持“与会者”快速 @，行动项直接转为待办或任务模块。
- 协作事项提供看板/列表双视图，支持拖拽状态、评论侧栏、提醒按钮。
- 时间轴可嵌在团队成员详情页、项目详情页，显示相关动态。

## 10. 风险与注意事项

- 权限控制要严密，尤其是包含敏感决策或人事信息的公告/纪要。
- 需避免重复劳动，和任务管理模块的边界要清晰（适合放置跨部门非标准事项）。
- 信息噪音控制：支持订阅管理、批量已读、关键词过滤，避免成员信息过载。
