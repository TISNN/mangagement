# 招聘与面试中心（Recruitment & Interview Hub）

## 1. 模块定位

招聘与面试中心承载从职位需求提出、候选人搜集、面试协作到 Offer 发放的全流程管理，并与“团队成员中心”的排班与权限配置形成前后衔接，帮助 HR 与用人部门统一掌握招聘进度、提升面试体验、沉淀数据资产，最终支撑人才决策。

## 2. 主要用户 & 角色

- **HR / 招聘专员**：发起职位需求、维护候选人库、协调面试、推动 Offer。
- **用人部门负责人 / 面试官**：提交岗位需求、参与面试、评估候选人。
- **候选人**：接收面试安排、提交资料、查询状态（未来候选人门户）。
- **招聘渠道合作方（猎头/高校/内推）**：提交候选人、查看反馈（可选只读权限）。

## 3. 核心能力

### 3.1 职位需求管理
- 提交职位需求表单：岗位信息、编制、预算、地点、优先级、任职资格。
- 自定义审批流：用人主管 → HRBP → 财务/高管（可配置）。
- 职位看板：展示开放中、暂停、已关闭的职位，支持筛选和搜索。
- 职位模板库：常用岗位快速复用 JD、流程与面试官配置。

### 3.2 候选人与渠道管理
- 候选人档案：基本信息、简历附件、技能标签、期望薪资、来源。
- 支持多渠道导入：手动录入、Excel 导入、邮箱解析、猎头提交链接。
- 管理候选人状态：待筛选 → 简历通过 → 面试中 → 发 Offer → 入职/淘汰。
- 候选人与多个职位的关联记录，支持转岗推荐。
- 渠道统计：各渠道投递量、面试率、Offer 率、入职率。

### 3.3 面试流程编排
- 面试流程模板：支持设置多轮面试（如 HR 初筛、专业面、主管面、试讲），并指派面试官。
- 面试安排：选择时间、地点/视频链接，自动同步日历邀请、发送提醒。
- 面试官排班：查看个人面试列表、确认/拒绝安排、标记冲突。
- 候选人通知：提供邮件/短信模板，包含时间、地点、注意事项。
- 资料与测评：上传技能测评题目、在线测评链接、性格测试等。

### 3.4 面试反馈与决策
- 面试反馈表：面试官填写评分、优缺点、录用建议、是否推荐下一轮。
- 支持打分维度（专业能力、沟通表达、文化契合等）与自定义评价项。
- 多面试官评分对比图，突出意见不一致的项。
- 决策流程：汇总反馈 → 招聘/用人主管决策 → Offer 审批（HR → 部门负责人 → 财务）。
- Offer 模板：薪资结构、福利、入职日期、试用期、其他条款。
- Offer 发放后自动生成待办提醒，引导 HR 在“团队成员中心”补充账号开通、设备发放等后续任务。

### 3.5 招聘统计与报表
- 招聘漏斗：按职位、部门、渠道展示投递 → 简历通过 → 面试 → Offer → 入职。
- 招聘周期：职位开放天数、平均面试轮次、Offer 接受率。
- 面试效率：面试官响应时效、面试完成率、迟到记录。
- 渠道效果：内推、猎头、校园招聘、社招网站的转化率对比。
- 支持导出 Excel / PDF 报表，供管理层与财务使用。

### 3.6 协作与提醒
- @提醒与评论：在候选人档案、面试反馈、职位需求中 @团队成员，支持附件。
- 审批提醒：职位审批、面试安排、Offer 审批逾期时的自动提醒（邮件/IM）。
- 面试官面板：面试排期、待填写反馈、评分历史、面试官绩效。
- 候选人模板：拒绝感谢信、录取通知、面试确认、资料补充提醒。

### 3.7 候选人体验（扩展）
- 候选人门户（下一阶段）：候选人查看面试安排、提交资料、确认 Offer。
- 自动发送面试前提醒、面试后感谢、录用/拒绝通知。
- 满意度调查：对招聘流程进行反馈，沉淀口碑。

## 4. 数据结构规划

| 表名 | 核心字段 | 说明 |
| --- | --- | --- |
| `recruiting_positions` | `id`, `title`, `department_id`, `location`, `type`（全职/兼职/实习）, `headcount`, `priority`, `status`, `description`, `requirements`, `opened_at`, `closed_at`, `owner_id` | 职位需求 |
| `recruiting_position_workflows` | `id`, `position_id`, `steps` JSONB（轮次名称、面试官、形式、提醒）, `version`, `is_active` | 面试流程定义 |
| `recruiting_candidates` | `id`, `name`, `email`, `phone`, `resume_url`, `expected_salary`, `current_company`, `skills[]`, `experience_years`, `source`, `channel_id`, `notes`, `privacy_consent` | 候选人档案 |
| `recruiting_applications` | `id`, `candidate_id`, `position_id`, `status`, `stage`, `applied_at`, `withdrawn_at`, `offer_id`, `owner_id`, `tags[]` | 候选人与职位申请 |
| `recruiting_interviews` | `id`, `application_id`, `stage`, `scheduled_at`, `end_at`, `location`, `meeting_link`, `interviewers[]`, `status`, `attachments[]` | 面试安排 |
| `recruiting_feedbacks` | `id`, `interview_id`, `reviewer_id`, `score`, `rating_details` JSONB, `recommendation`, `comment`, `submitted_at` | 面试反馈 |
| `recruiting_offers` | `id`, `application_id`, `status`, `salary`, `currency`, `bonus`, `equity`, `probation_months`, `start_date`, `approved_by`, `approved_at`, `sent_at`, `accepted_at`, `rejected_at` | Offer 记录 |
| `recruiting_channels` | `id`, `name`, `type`（猎头/内推/网站/校园）, `contact_info`, `notes`, `is_active` | 招聘渠道 |
| `recruiting_reports` | `id`, `type`, `filters`, `generated_by`, `file_url`, `generated_at` | 报表生成历史 |
| `recruiting_notifications` | `id`, `type`, `entity_id`, `recipient_id`, `sent_at`, `status`, `channel`, `payload` | 通知记录 |

## 5. 权限与视图

| 角色 | 权限概述 |
| --- | --- |
| **超级管理员（HR Manager）** | 配置流程、管理职位与渠道、查看所有候选人数据、导出报表 |
| **招聘专员（Recruiter）** | 创建职位、录入候选人、安排面试、提交 Offer、查看所属职位报表 |
| **面试官（Interviewer）** | 查看自己被邀请的面试、填写反馈、访问候选人简要资料（脱敏） |
| **用人部门负责人** | 审批职位需求、审批 Offer、查看本部门招聘进度与报表 |
| **渠道合作人** | 提交候选人、查看投递状态（仅自己的候选人，且需开启合作权限） |
| **合规/法务（可选）** | 查看候选人隐私协议、审核敏感操作日志 |

## 6. 与其他模块的衔接

- **团队成员中心**：Offer 接受后同步生成成员档案并预分配排班与权限，确保入岗准备。
- **协作与沟通记录**：招聘公告、面试纪要、跨部门协作事项同步到沟通 Hub。
- **绩效与目标看板**：招聘目标（如岗位填补率、招聘周期）可纳入 HR 团队 OKR。
- **排班与出勤洞察**：面试官排班冲突提醒，与已有排班数据结合。
- **外部系统**：支持导出到第三方薪酬系统或 ATS，对接日历、邮件、IM。

## 7. 指标与报表示例

- **招聘漏斗**：投递 → 简历筛选 → 面试 → Offer → 入职
- **职位效率**：开放天数、平均面试轮次、Offer 接受率、首月离职率（结合入职数据）
- **面试官表现**：反馈提交及时率、评分分布、推荐率
- **渠道表现**：各渠道候选人数量、面试通过率、成本对比
- **招聘成本**（后续拓展）：渠道费用、猎头佣金、面试成本（可选）

## 8. 实施路线图

1. **MVP**
   - 职位需求、候选人库、面试安排、反馈采集、简单漏斗报表。
   - 基本提醒：面试邀请、反馈提交、Offer 审批。
   - 手动导入/导出功能。

2. **增强版**
   - 自定义流程模板、渠道统计、面试官看板、Offer 模板与审批。
   - 候选人通信模板（邮件 / 短信）、面试冲突提醒、自动同步日历。
   - 报表导出 + 定时推送。

3. **成熟版**
   - 候选人门户、在线测评、AI 简历筛选、招聘计划预算管理。
   - 和第三方 ATS、HRIS、薪酬系统深度集成。
   - 招聘智能分析（供需预测、人才画像、面试质量评分）。

## 9. 风险与合规注意事项

- 保护候选人隐私（GDPR/本地数据法规），敏感信息需脱敏或限制访问。
- 明确数据留存策略（候选人未录用是否保留档案、保留多久）。
- 招聘决策记录可作为合规与风控凭证，需确保不可篡改。
- Offer 发放与薪酬数据需与财务对齐，避免信息偏差。

## 10. UI/交互建议

- 顶部导航包含“职位”、“候选人”、“面试日程”、“报表”、“设置”。
- 职位列表支持卡片视图（显示招聘进度）与表格视图（便于排序）。
- 候选人详情页展示时间轴（投递 → 面试 → 反馈 → Offer）。
- 面试日程可提供日历视图与列表视图，支持拖拽调度。
- 报表页面结合图表与指标卡，提供导出按钮与历史生成记录。
