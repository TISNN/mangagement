# 服务中心（Service Hub）设计方案

> 目标：面向合作留学机构与个人顾问，提供一站式 B 端服务（产品设计、开发、品牌营销、法律与礼品定制等），通过标准化流程、工单管理与服务包商品化，提升交付效率与收入多样性。

## 1. 模块定位
- **归属**：CRM Command Center / B2B Service Hub
- **用户角色**：
  - 客户成功经理 / 渠道经理：受理合作机构的需求、跟进交付
  - 服务专家（设计师、工程师、法务顾问、运营人员）：执行具体服务
  - 合作机构联系人：提交需求、审核方案、付款
  - 管理层：监控服务订单、盈利能力与满意度

## 2. 服务分类与商品化
| 分类 | 说明 | 典型服务包 | KPI |
| --- | --- | --- | --- |
| 数字化建设 | 网站、小程序、App 定制开发 | `Website Starter`、`Mini-app Launch`、`招生 CRM 集成` | 交付周期、上线质量、复购率 |
| 品牌营销 | 社媒运营、海报设计、宣传册、公众号推文 | `Social Media 30D Sprint`、`Brand Kit & Poster` | 内容产出量、互动率、CPC |
| 法律与合规 | 法律指导、合同模板、政策咨询 | `留学合同审核`、`留学政策法律顾问` | 响应时间、解决率 |
| 礼品与印刷 | 机构礼物、礼盒定制、宣传印刷 | `Welcome Gift Pack`、`协作礼盒` | 单均成本、满意度 |
| 运营支持 | 招生活动策划、数据分析、培训 | `Campaign Strategy`、`团队培训` | 活动 ROI、培训满意度 |

- 每个服务包包含：服务说明、交付项、时长、价格模式（套餐价/按小时）、样例成果、客户评价。
- 支持“定制请求”流程：客户描述需求 → 服务经理估价 → 生成报价方案。

## 3. 信息架构
```
Service Hub
├─ 概览 Dashboard
│  ├ KPI 总览（订单、收入、交付进度、满意度）
│  ├ 服务分类收益 & 热度排行
│  ├ 当前高优先级工单 / 风险提醒
├─ 服务目录 Catalog
│  ├ 服务包列表（标签、难度、交付时间）
│  ├ 服务详情页（含案例、交付流程、FAQ）
│  └ 自定义报价入口
├─ 工单与项目 Projects
│  ├ 工单列表：状态、负责人、客户、截止日期
│  ├ 工单详情：沟通历史、附件、任务分派、审批
│  └ 甘特 / 看板视图支持协同管理
├─ 客户门户 Client Portal
│  ├ 客户提交请求、查看报价、审批与付款
│  ├ 实时进度、交付文件、评价
│  └ 支持品牌化白标
├─ 资源与人才 Resource Pool
│  ├ 内部专家 / 外部合作伙伴档案
│  ├ 技能矩阵、可用时间、任务分配
├─ 合同与结算 Billing
│  ├ 订金/阶段付款管理
│  ├ 发票与凭证
│  └ 与财务模块对接
└─ 数据分析 Analytics
   ├ 收益、成本、工时分析
   ├ 客户满意度、NPS
   └ 服务包表现对比
```

### 3.1 前端页面架构（多级导航）
- `/admin/service-center` → 二级页面骨架，顶部标签导航。
  - `overview`：运营总览（指标卡、定制漏斗、SLA、风险提醒）。
  - `catalog`：服务目录（类目卡片、案例入口、AI 报价）。
  - `projects`：项目工单（工单表格、里程碑、作战手册）。
  - `resources`：资源池（专家档案、技能矩阵、利用率）。
  - `client-portal`：客户门户（客户动态、协作建议、知识沉淀）。
  - `analytics`：分析中心（收入结构、满意度、财务预测）。
- 支持未来扩展三级页面：例如 `projects/:orderId` 展示单个工单详情、审批流与文件日志。

## 4. 关键流程
### 4.1 标准服务包下单
1. 客户在目录选择服务包 → 查看说明/案例 → 点击“请求服务”。
2. 填写需求表单（背景信息、期望交付时间、预算）。
3. 系统生成工单 → 指派客户成功经理 (CSM)。
4. CSM 审核 → 若无需定制直接确认报价 → 客户支付订金。
5. 项目开始：任务分配给设计/开发/法务等专家。
6. 阶段交付：文件上传 → 客户审批 → 追加调整。
7. 最终交付与总结：客户评价、产出案例归档、收入确认。

### 4.2 定制化请求
- 增加“需求评估”步骤：
  - CSM 召开需求澄清会议 → 拆解任务 → 内部估算 → 生成报价文档。
  - 客户确认后进入项目管理流程。

### 4.3 服务生命周期管理
- 项目状态：`待确认` → `进行中` → `待验收` → `已完成` → `体验回访`。
- SLA：响应时间（如 12h 内响应），交付时间线（以甘特图展示）。
- 项目回顾：定期收集客户满意度与案例素材。

## 5. 数据 & KPI
| 指标 | 说明 |
| --- | --- |
| 服务订单数 / 成交率 | 按服务类别、客户类型统计 |
| 收入 / 毛利 / 成本 | 自动同步财务模块，计算利润率 |
| 项目交付进度 | 准时率、延期项、风险项 |
| 专家利用率 | 任务工时 vs 可用工时 |
| 客户满意度 / NPS | 来源于回访、评价 |
| 复购 / 交叉销售率 | 服务包与知识花园/课程联动 |

## 6. 系统模块联动
- **CRM**：线索转服务订单，客户联系人信息同步。
- **知识花园**：可将交付成果衍生知识产品（案例、模板）并上架。
- **财务模块**：合同、收款、成本、发票同步。
- **任务/项目管理**：与 Project Mission Board 共享项目视图和模板。
- **通知系统**：服务进度、审批、付款提醒（邮件、短信、站内）。

## 7. 权限与安全
| 角色 | 权限 |
| --- | --- |
| 管理员 | 配置服务包、定价、审批及人力资源分配 |
| CSM | 查看全部客户请求、创建报价、协调项目 |
| 服务专家 | 查看分配项目、上传交付物、记录工时 |
| 合作机构客户 | 仅访问自己项目与账单 |
| 外部合作伙伴 | 仅访问被授权的任务与文件 |
- 数据访问采用项目级权限，交付文件默认加水印与访问过期控制。

## 8. UI 草图（简述）
- **Dashboard**：左侧 KPI 卡 + 中部项目列表 + 右侧风险/提醒卡片。
- **Catalog**：服务包卡片（标题、标签、价格、时长），支持筛选（设计、开发、法务、营销、礼品）。
- **Project Detail**：顶部项目状态条、客户信息，主体为任务列表、讨论区、文件区、时间线。
- **Client Portal**：简洁白标风格，展示项目进度条、里程碑、付款状态与沟通历史。

## 9. 技术需求
- 后端：服务包、工单、任务、报价、工时、文件存储等数据模型；流程引擎实现审批和任务分配。
- 前端：React + Tailwind，采用现有组件库；集成甘特/看板组件；支持评论、富文本编辑、文件上传（带版本）。
- 集成：邮件/短信服务；文件存储（S3/OSS）；第三方法律咨询接口（可选）。

## 10. 迭代路线
1. **MVP**（2-3 Sprint）
   - 服务目录 + 工单管理 + 付款记录基础版本
   - CSM 指派流程 + 文件交付 + 客户门户查看
2. **阶段二**
   - 定价引擎（套餐+小时费率）、多阶段付款、甘特图
   - 专家资源池、工时记录、利用率报表
3. **阶段三**
   - 自动化模板（合同、报价单）、知识花园联动
   - 自动催款、智能推荐服务包（基于 CRM 数据）
4. **阶段四**
   - Marketplace 生态：支持第三方服务商入驻、结算分成
   - AI 助手：需求澄清、报价建议、工单优先级预测。

---
服务中心将成为机构对外 B2B 服务的核心枢纽，通过标准化产品＋项目交付管理，实现从需求收集到收益确认的闭环，并与现有 CRM、财务、知识体系模块深度集成。
