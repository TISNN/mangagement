# 学生管理中心（Student Management Center）设计文档

## 1. 模块定位与目标

- **所属导航**：`Starfleet Academy / Cadet Ledger`
- **角色覆盖**：顾问、运营、文书、客服、管理员。
- **核心目标**：
  - 统一管理学生全生命周期数据：基础信息、服务、导师、进度、风险。
  - 提供多视图切换（列表/卡片/表格/看板），满足不同角色的使用场景。
  - 打通服务工作台、项目任务、服务进度中心等模块，实现数据联动。

## 2. 顶层信息架构

```
学生管理中心
├─ 学生总览（Overview）
├─ 列表视图（Cadet List）
├─ 卡片视图（Galaxy Cards）
├─ 表格视图（Data Grid）
├─ 服务看板（Service Kanban）
├─ 学生详情（Detail Hub）
└─ 数据洞察（Insights）
```

### 2.1 学生总览（Overview）
- 指标卡片：学生总数、活跃人数、服务覆盖率、风险预警数。
- 过滤器：状态（活跃/休学/毕业/退学）、服务类型、顾问、标签。
- 统计组件：服务类型占比、来源渠道、区域分布、进度段位。

### 2.2 列表视图（Cadet List）
- 主要用于顾问/运营快速检索学生、查看服务摘要。
- 支持快速搜索、状态标签、服务数量、最新更新信息。
- 行操作：查看详情、分配顾问、添加服务、生成报告。

### 2.3 卡片视图（Card View）
- 更贴近 CRM 管理，展示头像、基础信息、服务进度条、导师团队、风险提示。
- 支持 hover 操作区：快速创建任务、发起沟通、查看服务时间线。
- 卡片上直接展示服务前两项，并提供展开更多。

### 2.4 表格视图（Data Grid）
- 面向数据运营角色，支持自定义列（联系人、学校、专业、服务、进度、收入等）。
- 支持多列排序、冻结列、导出 Excel。
- 可联动筛选条件，方便批量操作（如批量发送邮件）。

### 2.5 服务看板（Service Kanban）
- 按服务阶段（准备中/进行中/申请中/已完成/风险）展示学生，支持拖拽调整阶段。
- 每个卡片展示学生姓名、服务、顾问、进度百分比、风险标识。
- 支持按顾问或服务类型切换视图。

### 2.6 学生详情（Detail Hub）
- Tab 结构：`概览 | 服务项目 | 任务 | 文书 | 材料 | 网申 | 风险 | 档案`
- 概览：基础信息、服务摘要、导师团队、AI 生成的服务洞察。
- 服务项目：卡片/表格展示服务状态、里程碑进度（链接到服务进度中心）。
- 任务：关联项目任务控制台，显示当前任务、负责人、截止时间。
- 文书/材料/网申：嵌入工作台与网申进度模块的简版视图，支持跳转。
- 风险：展示当前风险项、等级、建议动作。
- 档案：历史记录、沟通纪要、文档附件。

### 2.7 数据洞察（Insights）
- KPI：转化率、服务完成率、满意度等指标。
- 学生分段：按来源、服务类型、阶段、风险等级进行聚类分析。
- 预测模型：AI 预测申请成功率或风险指数，辅助决策。

## 3. 核心功能详解

### 3.1 搜索与筛选
- 全局搜索：支持姓名、邮箱、手机、标签、服务关键词。
- 高级筛选：状态、服务类型、顾问、创建日期、国家、风险等级、标签。
- 保存筛选视图：用户可保存常用组合，快速切换。

### 3.2 批量操作
- 支持批量选择后执行：分配顾问、发送邮件/短信、导出数据、添加标签、生成报告。
- 批量操作有权限控制，需记录日志。

### 3.3 学生标签体系
- 标签类型：来源渠道、兴趣方向、服务类型、风险等级、语言能力等。
- 标签管理支持颜色、排序、别名配置。
- 支持自动标签：根据任务/风险/服务状态自动打标签。

### 3.4 AI 助手
- 学生洞察：自动生成学生服务摘要、风险提示、建议下一步动作。
- 服务回顾：一键生成阶段总结或家长汇报材料。
- 风险预警：识别多维数据（逾期任务、材料缺项、反馈）生成风险评分。

## 4. 数据模型（简化）

```
students
├─ student_profile (教育背景、目标、语言)
├─ student_service (服务记录)
│   ├─ service_progress (进度记录/里程碑)
│   ├─ service_tasks (任务联动)
│   └─ service_documents/materials/applications
├─ student_advisors (顾问/导师关联)
├─ student_tags (标签绑定)
└─ student_risk (风险记录)
```

关键字段：
- `students`: id, name, email, phone, status, source, country, created_at。
- `student_service`: service_type, status, start_date, end_date, progress, primary_advisor, linked_project_id。
- `student_risk`: level, score, summary, mitigation, detected_at, resolved_at。
- `student_tags`: tag_type, tag_value, confidence。

## 5. API 参考

- `GET /students`: 支持分页、搜索、筛选组合。
- `GET /students/:id`: 返回学生详情（支持模块化加载）。
- `POST /students`: 创建学生记录。
- `PATCH /students/:id`: 更新基础信息、状态、标签。
- `GET /students/:id/services`: 获取服务列表及进度。
- `GET /students/:id/insights`: 获取 AI 洞察、风险提示。
- `POST /students/batch-actions`: 执行批量操作。
- `GET /students/analytics`: 输出统计指标与报表数据。

## 7. 交互细节

- 列表行/卡片 hover 时展示快速操作按钮。
- 可将学生拖拽至“收藏夹”或“重点关注”区，自定义排序。
- 批量操作需二次确认，显示执行结果（成功/失败/部分成功）。
- 学生详情支持“在新窗口打开”或“复制链接分享”。
- 支持暗色模式与响应式布局。

## 8. 自动化与提醒

- 新增学生 → 自动生成欢迎任务、基础信息填充任务。
- 服务阶段变化 → 自动同步到服务进度中心。
- AI 风险评分超过阈值 → 发送提醒给顾问与运营。
- 学生长时间（>14天）无更新 → 自动弹出跟进建议。

## 9. KPI 与评估

- 学生数据完善率 ≥ 95%。
- 服务阶段更新及时率 ≥ 90%。
- 风险预警响应时长逐月下降。
- 学生满意度提升，投诉率下降。
- 顾问人均管理学生数增长但服务质量保持。

## 10. 迭代路线

1. **MVP**：列表/卡片视图 + 学生详情基础信息 + 服务摘要。
2. **视图增强**：表格自定义列、看板视图、批量操作。
3. **联动模块**：嵌入服务进度、项目任务、文书工作台等模块。
4. **智能化**：AI 洞察、风险预警、推荐下一步动作。
5. **数据分析**：上线学生画像、转化漏斗、服务绩效分析。
