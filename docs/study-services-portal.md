# 留学服务总览（Study Services Portal）设计文档

## 1. 模块定位与目标

- **所属导航**：`Starfleet Academy / Study Services Portal`
- **用户角色**：运营主管、教务经理、顾问团队负责人、行政支持。
- **核心目标**：
  - 提供一站式总览页面，串联学生、导师、服务进度、文书申请与任务等子模块。
  - 用数据指标、快捷入口和AI洞察帮助团队迅速判断业务运行状态。
  - 作为留学服务体系的起点，承担“入口 + 信息中枢”的双重角色。

## 2. 页面信息架构

```
留学服务总览
├─ 顶部概览（Hero Panel）
│  ├ 指标摘要卡片
│  ├ 快捷操作按钮
│  └ 个性化欢迎语 & 描述
├─ 指标看板（Metrics Board）
│  ├ 在读学生 / 活跃导师 / 服务进度均值 / 满意度 / 风险项目
│  └ 支持 hover 查看详细解释
├─ 模块速览（Submodule Tiles）
│  ├ 学生管理中心
│  ├ 导师管理中心
│  ├ 服务进度中心
│  ├ 文书与申请工作台
│  ├ 项目任务控制台
│  ├ 选校规划中心
│  └ 预留新模块入口
├─ AI 洞察（可迭代）
│  ├ 风险提示
│  ├ 优先任务
│  └ 建议下一步动作
└─ 最新动态（可选）
   ├ 最近风险项目
   ├ 近期重要里程碑
   └ 系统公告 / 培训通知
```

## 3. 关键组件设计

### 3.1 顶部概览（Hero Panel）
- **主要内容**：模块标题、副标题、说明文案。
- **快捷按钮**：如“前往学生管理”“查看任务看板”，采用主/次按钮组合。
- **视觉元素**：背景渐变 + 图标组合，突出品牌识别。

### 3.2 指标看板（Metrics Board）
- 列表示例：
  1. 在读学生数（显示新增趋势）
  2. 活跃导师占比
  3. 服务进度平均值
  4. 文书满意度均衡值
  5. 风险项目数量
- 点击指标可跳转到数据洞察模块或对应报表。
- 支持悬浮展示指标来源或环比计算方式。

### 3.3 模块速览（Submodule Tiles）
- 每张卡片包含：
  - 模块图标（统一图形语言）或渐变底纹。
  - 标题、副标题（“Starfleet Academy / 模块全称”）。
  - 核心描述（说明模块能力、适用场景）。
  - 标签（如“AI 洞察”“时间线”“排班”）。
  - “进入模块”按钮、查看详情按钮。
- 布局为 2~3 列响应式，支持 hover 阴影放大。

### 3.4 AI 洞察区（预留）
- 展示若干卡片：
  - 风险预警（如“质检导师过载”）
  - 关键任务（如“本周未更新服务进度的学生”）
  - 提升建议（如“推荐新增某类导师”）
- 按卡片格式呈现，每条含标题、说明、行动按钮。

### 3.5 最新动态区（可选）
- 组件示例：
  - 近期风险项目列表
  - 即将到期的里程碑或任务
  - 系统公告/培训通知
- 支持“一键查看全部”按钮，跳转到对应模块。

## 4. 交互体验

- 模块卡片、指标卡片默认可点击，支持键盘访问。
- 快捷按钮使用主/次色区分优先级。
- 使用轻量动画（hover、渐入）增强仪式感。
- 支持暗色模式：指标卡背景、模块卡图标自动适配。
- 页面加载后可加入欢迎提示、多次登录使用AI推荐内容。

## 5. 数据 & 接口建议

| 区块            | 数据来源 / 接口                               |
|-----------------|-----------------------------------------------|
| 指标看板        | `/api/dashboard/study-services/stats`         |
| 模块卡片        | 配置/常量 + usage metrics（可放在同接口返回） |
| AI 洞察         | `/api/dashboard/study-services/insights`      |
| 最新动态        | `/api/dashboard/study-services/activity`      |

- 数据结构建议：后端返回统一的Dashboard DTO，含指标、提示、卡片数据，减少前端请求。
- 支持缓存（短期）和实时刷新（按需触发）。

## 6. 技术实现建议

- 采用与其他管理页面一致的组件体系（Tailwind + 自定义卡片组件）。
- 指标卡提取为可复用组件，支持彩色图标、趋势和 tooltip。
- 模块卡使用 `motion`（可选）给 hover 动效。
- AI 洞察、最新动态可在 MVP 中预留占位（与现有页面一致）。
- 新增路由：`/admin/study-services`。
- 导航：一级菜单为“留学服务”，下含二级菜单（学生、导师、服务进度、文书工作台、项目任务、选校规划、旧版入口）。

## 7. 迭代路线

1. **MVP**：搭建页面骨架（指标卡 + 模块卡 + 占位）。
2. **数据对接**：接入指标 API，显示实时数据。
3. **AI 洞察**：接入 AI 服务，输出风险/任务建议。
4. **动态板块**：与服务进度、任务模块的提醒联动。
5. **个性化**：按角色定制卡片排序、默认快捷按钮。
