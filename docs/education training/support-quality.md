# Support & Quality 功能设计

## 1. 定位与目标
- 构建教学质量监控与服务保障体系，实现教学投诉、质检、改进闭环。
- 让学员体验、老师表现、教研优化形成数据驱动的持续提升机制。

## 2. 核心用户
- 质检团队 / 教学督导
- 班主任 / 学习教练
- 任课老师 / 教研负责人
- 客服 / 投诉处理专员
- 管理层（关注口碑、风险）

## 3. 业务流程
1. **反馈采集**：学员评价、家长反馈、老师自评、第三方调查。
2. **投诉处理**：登记 → 分派 → 跟进 → 解决 → 回访。
3. **质检执行**：听课抽检、作业抽查、资料审查。
4. **改进计划**：对表现问题生成整改任务、培训计划。
5. **复盘与沉淀**：形成案例库、优化教研与服务流程。

## 4. 功能清单
### 4.1 反馈与投诉管理
- 反馈来源：学员端、家长端、老师端、线下表单、客服热线
- 分类体系：教学质量、服务态度、课程安排、设施、财务
- SLA 管控：不同级别的响应时限、处理时限
- 处理流程：分派责任人、跨部门协作、处理记录、回访确认

### 4.2 质检任务
- 听课计划：随机抽检、重点班级、老师专项
- 评分维度：教学结构、课堂互动、专业度、课堂氛围
- 质检报告：得分、亮点、问题、改进建议
- 作业抽检：批改质量、反馈及时性、标准一致性

### 4.3 教师/班主任评价
- 学员与家长评价、匿名调查
- 老师自评与互评、班主任评价
- 评价结果汇总至教师绩效与培训计划

### 4.4 改进与培训
- 问题闭环任务：整改措施、负责人、截止时间
- 培训计划：针对薄弱项安排教研培训、观摩课
- 优秀案例分享：高分课堂、服务标杆

### 4.5 指标监控
- 投诉数量、类型、处理时长、满意度
- 质检覆盖率、平均得分、整改完成率
- 学员满意度趋势、NPS、口碑指数
- 老师培训完成率、问题复发率

## 5. 数据结构要点
- `FeedbackTicket`: {id, source, category, content, priority, status}
- `QualityInspection`: {id, targetType, targetId, scorecard, result}
- `ImprovementAction`: {id, issueId, assignee, dueDate, status}
- `TrainingPlan`: {id, focusArea, participants, schedule, outcome}
- `SatisfactionSurvey`: {id, audience, score, comments, period}

## 6. 集成需求
- 与 Learner/Tutor Portal 互通反馈入口与结果
- 与 CRM 同步投诉状态、回访记录
- 与 Progress & Evaluation 获取教学数据用于质检对比
- 与 HR/绩效系统同步老师质检结果、培训记录
- 与 Operations Dashboard 汇总风险与满意度指标

## 7. KPI 与监控
- 投诉处理 SLA 达成率、回访满意度
- 质检覆盖率、平均得分、整改及时率
- 学员满意度/NPS、问题复发率
- 教师培训与考核达成率

## 8. 迭代方向
- AI 智能分派：根据问题类型推荐责任团队与处理建议
- 情绪分析：语音/文本反馈的情绪识别与优先级判定
- 质检智能辅助：自动生成质检报告草稿、课堂亮点提取
- 建立知识库：常见问题、处理 SOP、优秀案例、培训资源
