# StudyLandsEdu 留学教育平台

## 项目简介

StudyLandsEdu是一个专业的留学教育管理平台，为留学机构提供全面的学生管理、服务跟踪、财务管理和资源库等功能。平台旨在提高留学服务的效率和质量，为机构、员工、导师和学生创建一个完整的服务生态系统。

## 核心功能

### 1. 人员管理
- **学生管理**: 完整的学生信息管理，包括基本信息、教育背景、目标国家等
- **员工管理**: 员工档案、技能、部门等信息管理
- **导师管理**: 专业导师资源管理，包括专业领域、收费标准等

### 2. 服务管理
- **服务类型**: 支持留学申请、留学咨询、语言培训、职场规划等多种服务类型
- **服务进度跟踪**: 记录服务的各个里程碑和进展情况
- **服务关联**: 实现学生与多种服务的关联管理

### 3. 财务管理
- **交易记录**: 管理收入和支出的完整记录
- **账户管理**: 支持多种账户类型，包括银行账户和电子钱包
- **财务分类**: 按项目、服务类型等维度进行财务分类

### 4. 聊天系统
- **公共频道**: 所有员工都可以查看和发送消息
- **私有频道**: 只有频道成员可以查看和发送消息
- **私聊会话**: 支持员工之间的一对一私聊
- **实时通知**: 基于Supabase实时订阅功能的消息通知

### 5. 学校和项目库
- **院校管理**: 全球院校信息管理，包括基本信息、排名、地区等
- **专业项目**: 各院校的专业项目信息，包括申请要求、课程等
- **成功案例**: 录取成功案例的记录和展示
- **收藏功能**: 允许用户收藏心仪的学校和项目

## 技术架构

### 前端
- React + TypeScript
- Vite 构建工具
- Tailwind CSS 样式框架
- 组件库：Radix UI

### 后端
- Supabase 后端服务
  - PostgreSQL 数据库
  - 用户认证与授权
  - 实时订阅功能
  - 存储服务

### 数据模型
系统包含多个核心数据模型：
- 人员管理：students, employees, mentors
- 服务管理：service_types, student_services, service_progress
- 财务管理：finance_accounts, finance_categories, finance_transactions, projects
- 聊天系统：chat_channels, chat_messages, channel_members, direct_messages, unread_messages
- 学校资源：schools, programs, success_cases, user_favorite_schools, user_favorite_programs

## 功能使用说明

### 学生管理
1. 添加新学生：录入基本信息、教育背景、目标国家等
2. 学生列表：查看所有学生，支持搜索和筛选
3. 学生详情：查看学生完整信息，包括关联的服务和交易记录

### 服务管理
1. 创建服务：选择学生、服务类型、分配导师
2. 进度更新：记录服务进展，添加里程碑、完成项目和下一步计划
3. 服务统计：查看不同类型服务的统计信息

### 财务管理
1. 记录交易：添加收入或支出记录，关联学生、服务或项目
2. 财务报表：生成按类别、时间等维度的财务报表
3. 账户管理：查看各账户余额和交易历史

### 聊天系统
1. 创建频道：创建公共或私有频道，添加成员
2. 发送消息：在频道中发送文本消息或附件
3. 私聊：与同事进行一对一私聊
4. 通知：接收新消息通知，标记已读/未读状态

### 学校和项目库
1. 浏览学校：查看全球各地区学校信息
2. 专业查询：搜索和筛选不同学校的专业项目
3. 收藏功能：收藏感兴趣的学校和项目，方便后续查看
4. 案例分享：查看和添加成功录取案例

## 数据同步与导入

系统提供多个数据导入工具：
- `import_programs.cjs`: 导入专业项目数据
- `save-australia-universities.cjs`: 导入澳大利亚大学数据
- `save-us-universities.cjs`: 导入美国大学数据
- `import_uk_programmes.cjs`: 导入英国专业项目数据

## 系统优化与注意事项

1. **权限控制**：系统使用Supabase的Row Level Security (RLS)策略控制访问权限
2. **数据备份**：定期备份数据库以防数据丢失
3. **性能优化**：大型列表使用分页加载，减少一次性加载的数据量
4. **实时更新**：使用Supabase的实时订阅功能，确保数据实时性
